# ET AI News

An AI-native news platform built for the ET AI Hackathon 2026 (Track 8).  
Submission by Avataar.ai x Economic Times, March 2026.

> "Business news in 2026 is still delivered like it's 2005 — static text articles, one-size-fits-all homepage, same format for everyone. Build something that makes people say 'I can't go back to reading news the old way.'"
> — PS-8 Problem Statement

---

## Overview

ET AI News replaces the static news homepage with a personalised, agent-driven intelligence layer. Instead of reading 8 separate articles across 15–20 minutes, users get a structured multi-angle briefing they can interact with in under 5 minutes.

The platform runs a three-step autonomous agent pipeline — Enrichment → Ranking → Synthesis — powered by LangGraph, with zero human input between steps.

## Architecture

The system is divided into three layers:

- **Frontend** — Next.js 14 App Router handles all rendering, routing, and state management.
- **Backend** — FastAPI exposes 12 REST endpoints and orchestrates the LangGraph agent pipeline. Attention signal processing runs as non-blocking background tasks.
- **Data Layer** — SQLite for structured storage, local spaCy/VADER for zero-cost NLP, Groq API for LLM inference.

### Agent Pipeline (LangGraph)

A stateful directed graph with three sequential nodes sharing a typed `NewsState`:

1. **Enrichment Agent** — Extracts named entities (ORG, PERSON, GPE, PRODUCT) via spaCy and scores sentiment via VADER. Category detection uses keyword scoring across 8 categories. All processing is local with no API calls.
2. **Ranking Agent** — Scores articles using a five-component relevance formula incorporating user category weights, topic preferences, entity interests, recency, and engagement signals. Articles scoring above 1.3 are flagged as personalised.
3. **Synthesis Agent** — Calls Groq 70B to generate structured briefings. For topic queries, produces 4 non-overlapping angles (Macro Impact, Market Reaction, Expert View, What to Watch).

If any node fails, enriched articles pass forward and the pipeline continues — no hard failures.

### LLM Routing

| Task | Model |
|---|---|
| Article enrichment fallback, story arc descriptions | llama-3.1-8b-instant |
| Briefing synthesis, chat, News Navigator | llama-3.3-70b-versatile |
| NER, sentiment scoring | Local (spaCy + VADER) — no API calls |

## Core Features

### Personalised Home Feed
Real articles from 9 RSS feeds displayed in an asymmetric bento-grid layout. Two modes: **General** (chronological) and **For You** (persona-ranked). Attention tracking runs silently in the background — dwell time, scroll depth, and click events update user `category_weights` via exponential moving average, re-ranking the feed each session.

### Article View with AI Briefing and Inline Chat
Each article page includes an AI briefing generated by the full LangGraph pipeline, plus a sticky right-side chat panel grounded in the specific article's content (headline, summary, entities, sentiment). Conversations can be carried to a full-page chat view via session storage.

### News Navigator
The platform's primary differentiator. Users enter any topic (e.g. "Union Budget", "RBI", "IPL 2026") and receive a structured 4-angle briefing synthesised from all matching articles. Each angle draws from different source articles. A right-side chat panel allows angle-specific follow-up questions. Trending topics in the sidebar are auto-generated from entity frequency across the last 50 articles.

### Story Arc Tracker
Groups articles by shared entities with 2+ mentions, building visual timelines for ongoing stories. Each arc includes a chronological dot timeline (colour-coded by sentiment), key players ranked by co-mention frequency, a sentiment trajectory (improving/declining/stable), and a 1-sentence forward prediction from Groq 8B.

### Authentication and Onboarding
Supabase handles email/password and Google OAuth. A 3-step onboarding modal collects topic preferences, trusted sources, and reading depth. Preferences seed initial `category_weights` so the "For You" feed is personalised from the very first session.

## Setup

### Prerequisites

- Node.js 18+
- Python 3.11+ (avoid 3.14 — LangGraph has Pydantic v1 compatibility issues)
- [Groq API key](https://console.groq.com) (free tier: 14,400 requests/day)
- [NewsAPI key](https://newsapi.org) (free)
- [Supabase project](https://supabase.com) (free)

### Backend

```bash
# Create and activate virtual environment
python -m venv venv && source venv/bin/activate

# Install dependencies
pip install fastapi uvicorn sqlalchemy pydantic python-dotenv requests feedparser \
    spacy vaderSentiment langchain-groq langgraph groq sentence-transformers

# Download spaCy model
python -m spacy download en_core_web_sm

# Configure environment
# Create backend/.env with:
# GROQ_API_KEY=xxx
# NEWS_API_KEY=xxx
# DATABASE_URL=sqlite:///./et_news.db

# Start server
cd backend && uvicorn main:app --reload --port 8000

# Populate database
python3 refresh_db.py
```

Swagger docs available at `http://localhost:8000/docs`.

### Frontend

```bash
cd frontend && npm install

# Create frontend/.env.local with:
# NEXT_PUBLIC_SUPABASE_URL=xxx
# NEXT_PUBLIC_SUPABASE_ANON_KEY=xxx
# NEXT_PUBLIC_API_URL=http://localhost:8000

npm run dev
```

Open `http://localhost:3000`. In Supabase, add `http://localhost:3000` as Site URL and `http://localhost:3000/**` as a Redirect URL.
